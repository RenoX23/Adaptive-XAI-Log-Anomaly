# 5. Experimental Setup

## 5.1 Datasets and Preprocessing
We evaluate AXEL on the industry-standard HDFS dataset \cite{xu2009detecting}, which comprises over 11.1 million log messages generated by a real-world Hadoop cluster. To ensure rigorous evaluation of structural sequence modeling, we eschewed simplistic line-by-line anomaly detection and instead parsed the logs using Drain \cite{he2017drain}, extracting 29 unique event templates. The logs were subsequently grouped by their native `BlockId` into complete execution traces. We curated a balanced empirical dataset of 10,000 un-truncated block traces (5,000 normal, 5,000 anomalous). The framework was trained exclusively on a subset of 3,500 normal sequences to learn the offline PCA manifold, while testing was conducted on a mixed set of 5,500 sequences (see Table I for full dataset characteristics).

## 5.2 Baselines
To demonstrate the architectural advantages of AXEL, we benchmarked against two widely adopted state-of-the-art deep learning log anomaly frameworks:
*   **DeepLog \cite{du2017deeplog}:** An LSTM-based model that treats log keys as natural language, predicting the next event iteratively.
*   **LogAnomaly \cite{meng2019loganomaly}:** An advanced LSTM that incorporates semantic vectorization. 
Both baselines were trained until convergence. For the Explainable AI (XAI) comparison, we benchmarked our Euler analytical method against **KernelSHAP** \cite{lundberg2017unified}, a state-of-the-art stochastic approximation method.

## 5.3 Evaluation Metrics
Following standard anomaly detection protocols, we evaluate model accuracy using Precision, Recall, and the F1-Score. Computational efficiency is measured via Inference Latency (milliseconds per sequence) to evaluate CI/CD deployment viability. Robustness to Concept Drift is quantified by calculating the relative F1-score improvement across sequential deployment epochs containing artificially injected distributional shifts.

# 6. Evaluation and Results

## 6.1 RQ1: Anomaly Detection Accuracy
Our primary evaluation assesses whether projecting positional-semantic embeddings onto a linear PCA manifold can rival the sequential learning capabilities of deep neural networks. As detailed in Table II, AXEL successfully outperforms both DeepLog and LogAnomaly. AXEL achieves a near-perfect Precision of 0.9940 and an F1-Score of 0.9905, compared to DeepLog's 0.9550 F1-Score. The results confirm that embedding sinusoidal positional vectors directly into Sentence-BERT tokens successfully flattens transition probabilities into spatial dimensions, capturing chronological anomalies without requiring recurrent hidden states. 

## 6.2 RQ2: Inference Latency and Throughput
The most critical bottleneck for industrial AIOps is inference latency. Figure 5 and Table II highlight the massive computational advantage of AXEL. While DeepLog requires $\sim$15.5 ms per sequence (due to iterative $O(N)$ LSTM operations) and LogAnomaly requires $\sim$17.2 ms, AXEL processes the same sequence in **0.015 ms**. By reducing inference to a single constant-time $O(d^2)$ projection on the residual matrix $C$, AXEL improves throughput by over 1,000$\times$, rendering it the only viable solution for high-velocity streaming environments.

## 6.3 RQ3: Interpretability and Root-Cause Analysis
When an anomaly is flagged, identifying the exact root-cause event is paramount. Table IV and Figure 4 compare AXEL's Euler feature attribution against KernelSHAP. For a test sequence containing an injected anomaly at $t_6$ (`Deleting block...`), KernelSHAP diffuses the attribution scores across the sequence, assigning a maximum weight of only 0.25 to the anomalous token. In contrast, AXEL’s Euler extraction analytically isolates the exact quadratic error, assigning an overwhelming 0.88 weight to $t_6$ while flattening benign tokens to near-zero. Furthermore, because AXEL calculates this gradient analytically ($c_i = x_i \cdot (C x)_i$), the XAI is generated instantly during the $O(1)$ forward pass, whereas SHAP requires hundreds of costly perturbation samples.

## 6.4 RQ4: Resilience to Concept Drift
To simulate the alert fatigue caused by routine software updates, we subjected the test stream to sequential drift epochs of escalating severity (Epoch 1: $0.5\sigma$ shift, Epoch 2: $1.0\sigma$ shift). As shown in Table III and Figure 3, a static PCA model suffers catastrophic degradation, with F1-Scores plummeting to 0.6376 as standard deviation shifts trigger false-positive cascades. However, AXEL’s ADWIN-backed dynamic thresholding successfully identifies the distributional drift. By triggering the 1D Gap-Finding algorithm, AXEL autonomously shifts the anomaly boundary $\tau$ to the new gap cluster, elevating the Epoch 2 F1-Score from 0.6877 to 0.7124. Over the full test stream, the zero-retraining adaptation mechanism yields a **+5.9% relative F1 improvement** (0.8451 vs 0.7979), successfully neutralizing alert fatigue.
